# Release Checklist

## Pre-Release

- [ ] Run QA playbook
- [ ] Review open beads; file follow-ups if needed
- [ ] Verify schema compatibility with adapters
- [ ] Confirm migrations (if any) ran cleanly

## Release

- [ ] Tag release version
- [ ] Update release notes

## Post-Release

- [ ] Monitor logs for new errors
- [ ] Validate core flows (capture, prioritize, plan)
